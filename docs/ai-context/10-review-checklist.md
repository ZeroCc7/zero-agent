# Review Checklist

本文件用于代码审查和前后端一致性检查。

## API 一致性

检查：

- 接口路径是否一致
- HTTP 方法是否一致
- 请求参数名称是否一致
- 请求参数位置是否一致：query / path / body
- 必填规则是否一致
- 响应结构是否一致
- 响应字段名称是否一致
- 枚举值是否一致
- 分页格式是否一致
- 时间格式是否一致
- 错误码是否一致
- 权限点是否一致

## 前端 Review

检查：

- 是否复用已有请求封装
- 是否处理 loading
- 是否处理 empty
- 是否处理 error
- 是否处理权限状态
- 表单是否有校验
- 删除等危险操作是否二次确认
- 是否有无关代码改动

## 后端 Review

检查：

- Controller 是否过重
- Service 是否承载业务逻辑
- Mapper / Repository 是否只负责数据访问
- 参数校验是否完整
- 权限校验是否完整
- 异常处理是否统一
- 事务边界是否合理
- SQL 是否存在明显性能问题
- 是否影响已有接口兼容性
