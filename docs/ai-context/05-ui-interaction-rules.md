# UI Interaction Rules

本文件定义前端 UI 和交互规范。

## 页面基本要求

每个页面需要明确：

- 页面标题
- 页面路由
- 页面模块
- 筛选区
- 操作区
- 表格或内容区
- 分页区
- 弹窗或抽屉
- 权限控制点

## 列表页要求

列表页默认包含：

- 查询条件
- 查询按钮
- 重置按钮
- 表格 loading
- 空数据状态
- 分页
- 操作按钮

常见操作：

- 新增
- 编辑
- 查看详情
- 删除
- 启用 / 禁用

删除类操作必须二次确认。

## 表单页要求

表单默认包含：

- 必填校验
- 格式校验
- 长度校验
- 提交 loading
- 取消操作
- 提交成功提示
- 提交失败提示

## 异常交互

- 接口失败：展示错误提示。
- 无权限：隐藏按钮或展示无权限提示。
- 无数据：展示空状态。
- 网络慢：展示 loading。
- 提交中：按钮禁用，防止重复提交。

## 文案规则

- 成功提示要明确。
- 错误提示要能指导用户下一步操作。
- 删除、禁用等危险操作必须明确后果。
