# Frontend Rules

本文件定义前端 Agent 的开发规范。

## 作用范围

前端 Agent 只负责：

- 页面开发
- 组件开发
- 路由配置
- API 请求封装
- 状态管理
- 表单校验
- 权限按钮展示
- loading / empty / error 状态
- UI 交互细节

前端 Agent 不应该修改后端代码。

## 开发前检查

开发前必须确认：

- 页面路由是什么
- 页面包含哪些模块
- 使用哪些 API
- API 字段是否已经在 `02-api-contract.md` 中定义
- 是否需要权限控制
- 是否需要表单校验
- 是否需要列表分页
- 是否需要新增组件
- 是否已有可复用组件

## API 使用规则

- 不允许凭空创造字段。
- 字段名必须严格匹配 API 契约。
- 枚举值必须严格匹配 API 契约。
- 如果接口契约缺失，先停止并提出契约补充建议。
- 如果接口返回字段和页面需求冲突，先报告冲突，不要私自适配。

## 页面状态要求

每个数据驱动页面默认需要处理：

- loading：请求中
- empty：无数据
- error：请求失败
- permission：无权限

## 表单要求

- 必填字段必须校验。
- 字符长度必须校验。
- 手机号、邮箱、数字等格式必须校验。
- 提交时必须防重复提交。
- 提交成功后根据业务刷新列表或关闭弹窗。
- 提交失败时展示错误信息。

## 组件复用规则

- 优先复用已有组件。
- 不要为一个页面写大量重复表格、表单、弹窗逻辑。
- 新组件必须命名清晰，职责单一。
- 不要引入新 UI 库，除非用户明确要求。

## 输出要求

开发完成后输出：

- 修改文件列表
- 使用的 API
- 使用的字段
- 权限点
- 自测步骤
