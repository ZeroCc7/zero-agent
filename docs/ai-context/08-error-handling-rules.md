# Error Handling Rules

本文件定义错误处理规范。

## 统一错误原则

- 后端返回统一错误结构。
- 前端使用统一错误展示方式。
- 错误信息应清楚说明原因。
- 不要把后端堆栈暴露给前端用户。

## 后端错误类型

常见错误：

- 参数错误
- 未登录
- 无权限
- 资源不存在
- 业务规则不允许
- 数据重复
- 系统异常

## 前端错误处理

- 400：展示参数错误信息。
- 401：跳转登录或触发重新登录。
- 403：展示无权限提示。
- 404：展示资源不存在。
- 500：展示系统异常提示。

## 输出要求

新增接口时，需要说明：

- 可能出现的业务异常
- 错误码
- message 示例
- 前端如何处理
