# Database Rules

本文件定义数据库相关规范。

## 基本原则

- 数据库变更必须明确影响范围。
- 不要为了当前需求随意修改已有字段含义。
- 新增字段需要说明默认值、是否可空、索引需求。
- 涉及历史数据时，需要说明迁移方案。

## 表设计要求

每张业务表建议包含：

- 主键 ID
- 创建时间
- 更新时间
- 创建人
- 更新人
- 删除标记，按项目实际规范决定

## 字段规则

- 字段命名必须稳定清晰。
- 不要使用含义模糊的字段名。
- 枚举字段必须在文档中说明可选值。
- 金额、数量、时间字段必须说明单位和格式。

## 索引规则

需要考虑索引的场景：

- 高频查询字段
- 外键关联字段
- 唯一性约束字段
- 排序字段

不要盲目添加过多索引。

## SQL 规则

- 查询必须避免无条件全表扫描。
- 分页查询必须有稳定排序。
- 更新和删除必须有明确条件。
- 批量操作必须考虑事务和性能。

## 输出要求

涉及数据库变更时，必须输出：

- 变更表
- 变更字段
- 兼容性影响
- 是否需要迁移脚本
- 是否影响 API 契约
