# Backend Rules

本文件定义后端 Agent 的开发规范。

## 作用范围

后端 Agent 只负责：

- API 实现
- Controller
- Service
- Repository / Mapper
- DTO / VO / Entity
- SQL
- 参数校验
- 权限校验
- 业务异常
- 事务控制
- 后端测试

后端 Agent 不应该修改前端代码。

## 开发前检查

开发前必须确认：

- API 是否已在 `02-api-contract.md` 中定义
- 请求字段是否明确
- 响应字段是否明确
- 数据库表是否存在
- 是否需要新增字段或索引
- 是否涉及权限
- 是否涉及事务
- 是否需要兼容已有接口
- 是否需要新增测试

## API 实现规则

- 路径、方法、请求参数、响应字段必须匹配 API 契约。
- 不允许私自修改响应结构。
- 不允许返回前端契约外的临时字段作为正式字段。
- 分页接口必须使用统一分页结构。
- 日期时间格式必须统一。

## 分层规则

- Controller 只做参数接收、参数校验入口和响应包装。
- Service 处理业务逻辑和事务边界。
- Mapper / Repository 只处理数据访问。
- DTO 用于请求入参。
- VO 用于接口响应。
- Entity 对应数据库表结构。

## 校验规则

- 必填字段必须校验。
- 字符长度必须校验。
- 枚举值必须校验。
- 业务唯一性必须校验。
- 权限相关接口必须做权限校验。

## 事务规则

涉及多表写入、状态变更、删除恢复、批量操作时，需要明确事务边界。

## 输出要求

开发完成后输出：

- 修改文件列表
- 实现的 API
- 数据库影响
- 权限影响
- 测试或验证步骤
