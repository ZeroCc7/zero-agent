# Testing Rules

本文件定义测试和验证规范。

## 基本原则

- 每次修改后运行最小相关验证。
- 如果无法运行测试，必须说明原因并给出手动验证步骤。
- 测试重点覆盖业务规则、异常分支和接口契约一致性。

## 前端验证

前端至少验证：

- 页面能正常打开
- 查询条件可用
- loading 状态正常
- 空数据状态正常
- 接口错误状态正常
- 表单校验正常
- 权限按钮显示正常

## 后端验证

后端至少验证：

- 参数校验
- 正常请求
- 异常请求
- 权限校验
- 分页格式
- 事务行为
- 数据库写入结果

## API 联调验证

检查：

- 请求路径
- HTTP 方法
- 请求参数
- 响应字段
- 枚举值
- 错误码
- 分页格式
- 时间格式
